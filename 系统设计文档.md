# 系统设计文档

## 一、项目概述

本系统基于 STM32F103C8T6 单片机，设计并实现了一款具备显示、6 轴传感器采集、蓝牙通信与计步功能的智能手表原型。系统由手表端固件与 Android 上位机两部分组成，已完整实现基础手表功能（时/分/秒）、传感器数据显示、页面切换以及与手机端的无线数据交互。

## 三、系统方案设计

### 3.1 系统总体架构

系统采用 **模块化分层设计**，总体分为以下几个模块：

- **主控模块**：STM32F103C8T6（72 MHz），负责系统控制与数据处理；
- **显示模块**：SSD1306 OLED，用于显示时钟、传感器数据、蓝牙状态、设备信息与活动统计；
- **传感器模块**：MPU6050，采集加速度、陀螺仪、温度数据，并支持姿态角计算；
- **通信模块**：HC-05 蓝牙（115200 baud DMA），与 Android 上位机进行双向数据交互；
- **时钟模块**：TIM2 1 Hz 中断驱动软 RTC，维护时/分/秒；
- **计步模块**：基于加速度向量幅值的低通滤波 + 动态阈值算法；
- **电源模块**：锂电池 + 3.3V 稳压，为整套系统供电。

### 3.2 硬件框图

```
┌──────────────┐
│  STM32F103C8T6  │  72 MHz 主控
│                 │
│  I2C1 ────────►  SSD1306 OLED   (0x3C, 128×64)
│  I2C2 ────────►  MPU6050        (0x68, 加速度+陀螺仪)
│  USART2 ─────►  HC-05 蓝牙      (115200 baud)

```

```

  ↓
I2C 总线恢复（防止从机死锁）
  ↓
HAL 初始化 + 时钟配置（72 MHz HSE）
  ↓

  ↓
OLED 初始化（SSD1306 命令序列）
  ↓

  ↓
软 RTC 变量初始化（00:00:00）
  ↓
计步器状态初始化
  ↓
┌──────────────────── 主循环（while(1)）────────────────────┐
│                                                          │
│  ├─ 每 200 ms：读取 MPU6050 → 更新计步器状态           │

```

## 四、详细设计文档

### 4.1 硬件部分

#### 4.1.1 电路原理设计

- **OLED (SSD1306)**：挂在 **I2C1** 上，SCL=PB6，SDA=PB7，7-bit 地址 `0x3C`；
- **MPU6050**：挂在 **I2C2** 上，SCL=PB10，SDA=PB11，7-bit 地址 `0x68`；
  > 将 OLED 与 MPU6050 分在两条 I2C 总线上，避免单一总线故障导致整套系统停摆；
- **HC-05**：接 **USART2**（PA2=TX，PA3=RX），STATE 引脚接 **PB0**（输入，上拉），波特率 115200；
- **TIM2**：内部时钟 72 MHz，预分频 7199（得到 10 kHz），周期 9999（得到 1 Hz），用于 RTC tick；
- **DMA1 通道 6**：用于 USART2 RX 环形缓冲，配合 Idle 中断实现非阻塞接收；
- **电源**：锂电池 → AMS1117-3.3V → 为 STM32 / OLED / MPU6050 / HC-05 统一供电。

#### 4.1.2 关键元件参数

| 器件 | 参数 |
|------|------|
| STM32F103C8T6 | Cortex-M3，72 MHz，64 KB Flash，20 KB RAM |
| SSD1306 | 128×64 单色（黄蓝双色）OLED，I2C 接口，3.3V |
| MPU6050 | 加速度 ±2g（16384 LSB/g），陀螺仪 ±250°/s（131 LSB/°/s），内置温度传感器 |
| HC-05 | 经典蓝牙 SPP v2.0+EDR，波特率 9600/19200/38400/**115200** 可配 |

#### 4.1.3 PCB 布局要点

- I2C 的 SCL/SDA 走线尽量短，减少寄生电容；
- MPU6050 放置在 PCB 靠近中心位置（振动/加速度读数更准）；
- HC-05 天线区域避开铜箔和金属元件；
- 电源与 GND 铜箔宽度 ≥ 2 mm；
- 锂电池接口靠近充电模块，远离 MCU / 传感器数字区域。

#### 4.2.3 关键算法与数据结构

**软 RTC**：
```c
typedef struct { uint8_t hour; uint8_t min; uint8_t sec; } RTC_Time;
// TIM2 1Hz 中断内：sec++ → 满 60 进 min → 满 60 进 hour → 满 24 归零
```

**计步器**：
- 输入：MPU6050 原始加速度（每 200 ms 采样一次）；
- 处理：`mag = √(ax² + ay² + az²)` → 低通滤波 `α=0.1` → 与动态阈值 `1.05g` 比较；
- 规则：超过阈值且与上次检测间隔 ≥ 200 ms 则计一步；
- 输出：步数、距离（步长 0.7 m × 步数）、卡路里（0.04 × 步数）。

**蓝牙帧协议**：
```
┌──────┬─────┬──────────────────────────────────┬──────┬──────┐
│ STX  │ CMD │             DATA                  │ CHK  │ ETX  │
│ 0xAA │     │ (小端 float / 8-bit / 字符串)     │ XOR  │ 0x55 │
└──────┴─────┴──────────────────────────────────┴──────┴──────┘
  CMD=0x01: 传感器上报 → 6 × float (ax, ay, az, gx, gy, gz)
  CMD=0x02: 时间同步   → 3 × uint8 (hour, min, sec)
  CMD=0x03: 计步数据   → 1 × uint32 (steps) + 1 × float (distance)
```

#### 4.2.4 外设配置方案（已实际写入 CubeMX）

| 外设 | 模式 | 参数 |
|------|------|------|
| **I2C1** | 标准模式 | 100 kHz，7-bit 地址，SCL=PB6，SDA=PB7 |
| **I2C2** | 标准模式 | 100 kHz，7-bit 地址，SCL=PB10，SDA=PB11 |
| **USART2** | 异步 + DMA | 115200 baud，8-N-1，TX=PA2，RX=PA3 |
| **TIM2** | 基本定时 | PSC=7199，ARR=9999 → 1 Hz 中断 |
| **DMA1 Channel 6** | 循环模式 | USART2 RX，内存 → 环形缓冲 |
| **PB0** | GPIO 输入 | 上拉，读取 HC-05 STATE 引脚状态 |

#### 4.2.5 错误处理与恢复机制

- **I2C 从机死锁**：启动时先执行总线恢复（SCL 发 9 个时钟脉冲 → SDA 发 STOP）；
- **MPU6050 WHO_AM_I 失败**：标记 error 位，主循环每 3 秒重新尝试初始化；
- **HC-05 断开**：通过 STATE 引脚电平（低=未连接）自动停止数据上报，避免无效发送；
- **蓝牙帧校验错误**：丢弃当前帧，环形缓冲指针回滚以等待下一个合法 STX。

## 六、结论

本项目以 STM32F103C8T6 为主控，融合 OLED 显示、MPU6050 6 轴传感器、HC-05 蓝牙通信、软 RTC 与计步器算法，并配套 Android BLE 上位机。项目分 Phase 0-5 逐步推进，最终在 `Phase3_Bluetooth/` 中完成全功能整合，系统稳定运行，具备较强的教学实践意义与扩展潜力。

后续可考虑的扩展方向：
1. 将按键/旋转编码器接入 GPIO，实现手动翻页（替代当前的 3 秒自动切换）；
2. 将 FreeRTOS 引入，将显示、传感器、蓝牙、RTC 拆分为独立任务；
3. 增加心率传感器（MAX30102）或环境光传感器；
4. 将 HC-05 替换为 HM-10（BLE 4.0），进一步降低功耗。
