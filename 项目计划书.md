## 二、项目背景与目标

### 2.1 项目背景

随着嵌入式系统与物联网技术的发展，智能穿戴设备已成为学习与应用开发的重要方向。基于 STM32 单片机、OLED 显示、MPU6050 6 轴传感器以及 HC-05 蓝牙通信，本项目设计并实现了一款具备显示、检测与无线通信功能的智能手表原型系统，并配套 Android 上位机用于数据交互与时间同步。

### 2.2 项目目标（已全部达成）

- ✅ 完成手表系统硬件平台的搭建与调试（STM32F103C8T6 + OLED + MPU6050 + HC-05 + 锂电池）；
- ✅ 实现 OLED 显示（5 个页面循环切换，含大字时钟）、传感器数据采集（6 轴物理量 + 姿态角 + 温度）与蓝牙通信（DMA 环形缓冲 + 帧协议）；
- ✅ 实现 Android/PC 上位机与手表端的数据交互（Android APP 已签名，可直接安装）；
- ✅ 实现软 RTC（TIM2 1Hz 中断）与计步器算法（加速度向量幅值 + 低通滤波 + 动态阈值）；
- ✅ 形成完整的项目文档（README + 系统设计文档 + 本计划书）与演示成果。

## 六、风险分析与应对（实际遇到的问题）

### 6.1 已识别风险 + 实际解决情况

| 风险 | 描述 | 应对措施 | 结果 |
|------|------|----------|------|
| I2C 从机死锁 | OLED/MPU6050 上电时 SDA/SCL 可能处于异常状态，导致 MCU 初始化超时 | 启动时先执行 **I2C 总线恢复**（SCL 发 9 个时钟脉冲 + SDA 发 STOP），然后再初始化外设 | ✅ 已在 `i2c_drv.c` 中实现，系统启动可靠 |
| MPU6050 WHO_AM_I 校验失败 | 焊接/接触/总线噪声导致 WHO_AM_I 读回非 0x68 | 标记 error 位，主循环每 3 秒重新尝试初始化，避免一次失败导致传感器永久停摆 | ✅ 已在 `mpu6050.c` + `main.c` 中实现 |
| HC-05 波特率不匹配 | HC-05 默认 9600 baud，但固件使用 115200 | Phase 0 阶段通过 AT 指令 `AT+UART=115200,0,0` 统一配置，所有文档与代码约定默认 115200 | ✅ 已稳定 |
| 蓝牙帧截断/噪声 | 无线链路噪声导致帧 STX/ETX 不完整或 CHK 失败 | 使用 DMA 环形缓冲 + Idle 中断，逐字节扫描 STX，遇到校验错误直接丢弃当前帧等待下一个合法 STX | ✅ 已在 `bluetooth.c` 中实现 |
| 时间无法自动更新 | 原版本无 RTC，时间变量始终为初始值 | 引入 TIM2（PSC=7199, ARR=9999 → 1 Hz），在中断内递增时/分/秒，手机端蓝牙可同步时间 | ✅ 已在 `soft_rtc.c/h` + `tim.c` + `stm32f1xx_it.c` 中实现 |
| 计步器误计 | 静止状态抖动/噪声可能被误判为一步 | 使用加速度向量幅值 + 低通滤波（α=0.1） + 动态阈值（1.05g） + 200 ms 最小间隔防抖 | ✅ 已在 `pedometer.c/h` 中实现 |
| SmartWatch2 与 Phase 项目功能分叉 | 两个版本（同伴写的 SmartWatch2 与本组 Phase 0-5）同时存在功能重叠 | 决定以 Phase 项目为主线，将 SmartWatch2 的 RTC/总线恢复/计步器特性反向合并到 `Phase3_Bluetooth/`，然后删除 SmartWatch2 目录 | ✅ 已完成，单一主项目 |
| Android 与手表端协议不一致 | 两端独立开发可能导致 STX/ETX/CMD/字节序不匹配 | 统一约定 `0xAA` / `0x55` 作为帧头帧尾，XOR 校验，小端 float，两端实现对称一致 | ✅ 已验证，连接后可正常收发 |

### 6.2 遗留的潜在问题与后续建议

- **HC-05 经典蓝牙功耗较高**：若需长期佩戴可考虑换成 HM-10（BLE 4.0）；
- **无按键交互**：当前页面为 3 秒自动切换，如需要手动控制可接入旋转编码器或按键；
- **未引入 RTOS**：当前为超级循环（`while(1)` + 时间戳调度），模块增加后可考虑迁移到 FreeRTOS；
- **未做低功耗优化**：72 MHz 全速运行 + OLED 始终点亮，续航偏短，后续可做间歇显示与 STOP 模式。

## 七、结论

本项目围绕智能手表嵌入式系统进行设计与实现，通过分阶段（Phase 0-5）开发完成了 OLED 显示驱动、MPU6050 6 轴传感器采集、HC-05 蓝牙通信（含 Android 上位机）、软 RTC、计步器算法以及 5 页面 UI 等全部功能。项目过程中遇到的 I2C 死锁、MPU6050 初始化失败、蓝牙波特率不匹配、计步误计等问题均已给出可复用的解决方案。

当前 `Phase3_Bluetooth/` 已作为统一主项目完成所有功能整合，SmartWatch2 目录已删除。通过本项目，三位成员分别加深了对 STM32 开发、外设驱动、蓝牙通信、嵌入式系统设计流程、Android 上位机以及项目文档编写的理解。
