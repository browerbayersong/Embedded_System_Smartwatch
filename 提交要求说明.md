# 课程设计提交要求说明

## 1. 提交计划书

时间：2026-06-30 9:00

各组提交《项目计划书》（本阶段为小组任务，每组提交一份），内容包括：

- 工作计划：从选题到验收的详细时间节点，明确各阶段里程碑
- 时间安排：组内成员的可投入时间与分工协调计划
- 物料采购计划：列出所需全部物料清单（型号、数量、单价、采购渠道），预估总成本
- 分工方案：明确每位成员的具体职责（硬件设计、软件开发、测试调试、文档撰写等）

### 建议提交格式

1. 封面：项目名称、组名、成员姓名、学号、指导老师
2. 目录
3. 项目背景与目标
4. 工作计划与时间节点
5. 物料清单与预算
6. 组内分工与责任划分
7. 风险分析与应对措施

---

## 2. 设计文档提交与汇报

时间：2026-07-02 9:00

各组提交《系统设计文档》并分组汇报（本阶段为小组任务，每组提交一份），内容包括：

- 物料到位情况：已到货物料清单与实物核对情况，未到货物料的预计到位时间
- 系统方案设计：系统总体架构、硬件框图、软件流程图
- 详细设计文档：
  - 硬件部分：电路原理图、关键元件参数计算、PCB布局规划（如有）
  - 软件部分：模块划分、关键算法描述、外设配置方案、任务调度设计（FreeRTOS任务划分等）
- 进度汇报：当前完成情况与后续工作计划

### 建议提交格式

1. 系统总体设计
2. 硬件设计说明
3. 软件设计说明
4. 进度与风险说明
5. 后续开发计划
6. 汇报PPT或答辩材料

---

## 3. 推荐提交注意事项

- 文档内容要逻辑清晰、结构完整，避免仅写概述。
- 物料清单尽量包含型号、数量、价格和采购渠道。
- 设计文档建议配图，尤其是硬件框图、软件流程图和系统结构图。
- 汇报时建议重点说明：当前完成什么、还差什么、下一步怎么做。
