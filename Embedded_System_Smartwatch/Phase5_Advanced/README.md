# Phase 5：高级功能

## 目标

在 Phase 4（FreeRTOS 稳定运行）的基础上，添加计步算法、多级菜单系统、电源管理等增强功能。

**本阶段是纯软件开发，不涉及硬件连接变更和 CubeMX 改动。**

## STM32CubeMX 配置

**无需改动。** Phase 4 的配置已包含所有需要的外设（ADC 在 Phase 4 已配置 PA0，本阶段启用即可），直接写代码。

## 本阶段新增/修改内容

### 5.1 计步算法

**修改文件：** `taskSensor` 任务函数

**算法：** 峰值检测 + 动态阈值

```
每 20ms 执行一次：
  1. 读取三轴加速度 Ax, Ay, Az
  2. 计算合成加速度 accel_mag = sqrt(Ax² + Ay² + Az²)
  3. IIR 低通滤波：filtered = 0.1 × accel_mag + 0.9 × filtered_prev
  4. 动态阈值：threshold = (max_val + min_val) / 2 （每 50 采样更新）
  5. 峰值检测：
     - filtered > threshold 且之前低于阈值 → 检查步间隔
     - 步间隔 > 200ms → 步数+1
     - filtered < threshold → 重置检测状态
  6. 发送步数到显示任务
```

**关键参数：**

| 参数 | 值 | 说明 |
|------|-----|------|
| α 滤波系数 | 0.1 | 越小滤波越强，延迟越大 |
| 最小步间隔 | 200ms | 最快 300 步/分 |
| 动态阈值窗口 | 50 采样 | 约 1 秒 |
| 基础阈值 | 1.1g | 低于此值不触发 |

> **调试技巧：** 用蓝牙把 `filtered` 和 `threshold` 实时传出来，电脑上画波形图。步行时加速度应有规律波峰（1.2g~2.0g），根据波形调参。

### 5.2 多级菜单系统

**修改文件：** `taskMenu` 任务函数 + `taskDisplay` 渲染逻辑

**菜单树结构：**

```
主菜单
├── 1. 时间显示
│   ├── 当前时间
│   ├── 日期
│   └── 步数统计
├── 2. 传感器数据
│   ├── 姿态角度 (Pitch/Roll/Yaw)
│   ├── 加速度 (X/Y/Z)
│   └── 温度
├── 3. 运动数据
│   ├── 今日步数
│   ├── 运动距离
│   └── 卡路里估算
├── 4. 设置
│   ├── 时间设置
│   ├── 蓝牙配对
│   ├── 屏幕亮度
│   └── 恢复出厂
└── 5. 系统信息
    ├── 电池电压
    ├── 固件版本
    └── 蓝牙 MAC
```

**编码器操作映射：**

| 操作 | 菜单层 | 设置层 |
|------|--------|--------|
| 顺时针 | 光标 ↓ | 数值 + |
| 逆时针 | 光标 ↑ | 数值 - |
| 短按 | 进入 | 确认 |
| 长按 | 返回 | 取消 |

**实现思路：**

定义 `MenuState` 结构体记录当前层级、光标位置和页面索引。TIM3 CNT 变化量判断旋转，EXTI10 中断时间戳判断短/长按。通过 `queueMenuMsg` 传递操作，`taskDisplay` 根据状态渲染对应页面。

### 5.3 电源管理

**ADC 电池电压读取（PA0，已在 Phase 4 配置）：**

```c
// 电池电压 = ADC值 / 4096 × 3.3V × 2（分压比）
HAL_ADC_Start(&hadc1);
uint16_t adc_val = HAL_ADC_GetValue(&hadc1);
float battery_v = (float)adc_val / 4096.0f * 3.3f * 2.0f;
```

**低功耗策略：**

| 状态 | 触发 | 动作 | 功耗 |
|------|------|------|------|
| 正常运行 | 编码器有操作 | 全速 72MHz | ~30mA |
| 熄屏 | 10 秒无操作 | OLED 发送休眠命令 (0xAE) | ~15mA |
| 一键关机 | 长按 3 秒 | PB12 拉低 → PMOS 截止 | 0 |

## 验证标准

- 走路时计步数据准确（100 步误差 < 5 步）
- 多级菜单切换流畅，编码器操作响应及时
- 熄屏后旋转编码器可唤醒
- 电池电压读数正确（与万用表对比）
- 所有 Phase 1~4 功能保持正常

### 常见问题

| 问题 | 检查项 |
|------|--------|
| 计步偏多 | 阈值过低 → 提高基础阈值或增加滤波 |
| 计步偏少 | 阈值过高 → 降低阈值，检查步行时 filtered 峰值 |
| 菜单跳变 | 编码器 CNT 读后未清零，或旋转速度过快 |
| 熄屏后唤不醒 | OLED 休眠后 I2C 是否仍可正常通信 |
| 电池读数不准 | 分压电阻阻值是否有偏差，校准系数 |
