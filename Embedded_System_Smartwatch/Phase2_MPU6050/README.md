# Phase 2：接入 MPU6050 陀螺仪

## 目标

在 Phase 1（OLED 可用）的基础上新增 MPU6050，读取加速度和角度数据并显示在屏幕上。

## 硬件连接

**MPU6050 使用独立 I2C2 总线，不与 OLED 共用。** 每路 I2C 只需 4 根杜邦线，接线干净可靠。

```
OLED (I2C1):                    MPU6050 (I2C2):
  PB6  — SCL                      PB10 — SCL
  PB7  — SDA                      PB11 — SDA
  3.3V — VCC                      3.3V — VCC
  GND  — GND                      GND  — GND
```

| 设备 | I2C 外设 | SCL | SDA | 7位地址 |
|------|---------|-----|-----|---------|
| SSD1306 OLED | I2C1 | PB6 | PB7 | 0x3C |
| MPU6050 | I2C2 | PB10 | PB11 | 0x68 |

> 两条 I2C 总线完全独立，互不干扰。适合杜邦线连接，无需并线/分线。

## MPU6050 详细介绍

### 概述

MPU6050 是 InvenSense 公司推出的 **6 轴运动传感器**，内部集成了：

- **3 轴陀螺仪**（角速度）：±250 / ±500 / ±1000 / ±2000 °/s 四档可调
- **3 轴加速度计**（线性加速度）：±2g / ±4g / ±8g / ±16g 四档可调
- **数字运动处理器（DMP）**：内置姿态解算引擎，可直接输出四元数
- **温度传感器**：芯片温度检测
- **I²C 主控制器**：可外挂一个第三方传感器（如磁力计），组成 9 轴系统

芯片内部通过 16 位 ADC 将模拟量转为数字量，经 DLPF（数字低通滤波器）后存入寄存器，MCU 通过 I²C 读取。

### 主要参数

| 参数 | 规格 |
|------|------|
| 工作电压 | 2.375V ~ 3.46V |
| 通信接口 | I²C（最高 400kHz Fast Mode） |
| I²C 地址 | AD0=GND → 0x68 / AD0=VCC → 0x69 |
| 加速度计量程 | ±2g（默认）/ ±4g / ±8g / ±16g |
| 陀螺仪量程 | ±250°/s（默认）/ ±500°/s / ±1000°/s / ±2000°/s |
| 加速度计灵敏度 | 16384 LSB/g（±2g 档） |
| 陀螺仪灵敏度 | 131 LSB/(°/s)（±250°/s 档） |
| 采样率 | 最高 1kHz（由 SMPLRT_DIV 和 DLPF 共同决定） |
| DLPF 带宽 | 5Hz / 10Hz / 21Hz / 44Hz / 94Hz / 184Hz / 260Hz |
| 数据格式 | 16 位有符号整数（补码），大端序 |
| FIFO | 1024 字节 |
| 功耗 | 正常工作 ~3.9mA，待机 ~5μA |

### 引脚说明

| 引脚 | 方向 | 功能 |
|------|------|------|
| VCC | 电源 | 3.3V 供电 |
| GND | 电源 | 接地 |
| **SCL** | 输入 | I²C 时钟线，接 MCU 的 SCL（本项目：PB10） |
| **SDA** | 双向 | I²C 数据线，接 MCU 的 SDA（本项目：PB11），需 4.7kΩ 上拉至 VCC |
| **AD0** | 输入 | I²C 地址选择：接 GND → 0x68，接 VCC → 0x69。同一条 I²C 总线上挂两个 MPU6050 时用来区分 |
| **INT** | 输出 | 可编程中断，开漏输出，需外接上拉。可配置为：数据就绪（Data Ready）、FIFO 溢出、运动检测（Motion Detect）、自由落体检测等 |
| **XDA** | 双向 | 辅助 I²C 数据线（主机模式），用于外接磁力计等第三方传感器 |
| **XCL** | 输出 | 辅助 I²C 时钟线（主机模式） |

> 本项目只用了 VCC、GND、SCL、SDA、AD0 五个引脚。AD0 接地（地址 0x68）。INT、XDA、XCL 悬空不用。

### 内部架构

```
                    ┌──────────────────────────────┐
   Accelerometer ──►│                              │
     3-axis         │   16-bit ADC  ──►  DLPF      │
                    │                              │
   Gyroscope   ──►│                              │──► FIFO (1024B) ──► I²C Slave
     3-axis         │   16-bit ADC  ──►  DLPF      │                   (to MCU)
                    │                              │
   Temperature ──►│   16-bit ADC                  │
                    │                              │
                    │   DMP (Digital Motion        │
                    │        Processor)            │
                    │   姿态解算 / 四元数输出       │
                    │                              │
   Aux I²C Master   │   XDA / XCL                 │──► 外部磁力计
                    │   (外挂传感器接口)             │
                    └──────────────────────────────┘
```

核心数据流：

1. MEMS 传感器 → 模拟电压
2. 16 位 ADC → 数字量
3. DLPF（数字低通滤波器）→ 去噪
4. 存入数据寄存器（ACCEL_XOUT_H 等）
5. MCU 通过 I²C 读取

DMP（Digital Motion Processor）是 MPU6050 的亮点功能：它**在芯片内部**完成姿态解算，直接输出四元数或欧拉角，不占用 MCU 算力。但 DMP 需要加载固件（约 3KB），本项目为透明起见，改为在 MCU 端通过加速度计数据直接计算俯仰角和横滚角。

### 坐标系

```
        +Z (up)
         |
         |
         *--- +X (forward, toward the dot on chip)
        /
       /
     +Y (left)
```

- **X 轴**：芯片表面有圆点标记的一侧方向
- **Y 轴**：与 X 在水平面内正交，左手定则
- **Z 轴**：垂直芯片表面向上

平放时：ax≈0, ay≈0, **az≈+9.81 m/s²**（重力加速度在 Z 轴上）。

### 量程与灵敏度对照

#### 加速度计

| ACCEL_CONFIG[4:3] | 量程 | 灵敏度 (LSB/g) |
|-------------------|------|----------------|
| 0 0（默认） | ±2g | 16384 |
| 0 1 | ±4g | 8192 |
| 1 0 | ±8g | 4096 |
| 1 1 | ±16g | 2048 |

#### 陀螺仪

| GYRO_CONFIG[4:3] | 量程 | 灵敏度 (LSB/(°/s)) |
|------------------|------|-------------------|
| 0 0（默认） | ±250°/s | 131 |
| 0 1 | ±500°/s | 65.5 |
| 1 0 | ±1000°/s | 32.8 |
| 1 1 | ±2000°/s | 16.4 |

> 本项目使用默认量程。加速度转换：`a = raw / 16384 × 9.81`（m/s²）。陀螺仪转换：`ω = raw / 131`（°/s）。

### 常用寄存器一览

| 寄存器 | 地址 | 位宽 | 说明 |
|--------|------|------|------|
| **WHO_AM_I** | 0x75 | 8 | 设备 ID，固定 0x68（上电默认） |
| **PWR_MGMT_1** | 0x6B | 8 | bit6=睡眠，bit3=温度传感器禁用。写 0x00 唤醒 |
| **SMPLRT_DIV** | 0x19 | 8 | 采样率 = 1kHz / (1 + DIV)。写 4 → 200Hz |
| **CONFIG** | 0x1A | 8 | bits[2:0]=DLPF_CFG，0x03 → 44Hz（加速度）/ 42Hz（陀螺仪） |
| **GYRO_CONFIG** | 0x1B | 8 | bits[4:3]=量程选择 |
| **ACCEL_CONFIG** | 0x1C | 8 | bits[4:3]=量程选择 |
| **ACCEL_XOUT_H/L** | 0x3B~0x40 | 16×3 | 加速度 X/Y/Z 原始值（大端序） |
| **TEMP_OUT_H/L** | 0x41~0x42 | 16 | 温度原始值：`T = raw/340 + 36.53` °C |
| **GYRO_XOUT_H/L** | 0x43~0x48 | 16×3 | 陀螺仪 X/Y/Z 原始值（大端序） |
| **INT_ENABLE** | 0x38 | 8 | 中断使能：bit0=Data Ready |
| **INT_STATUS** | 0x3A | 8 | 中断状态标志 |
| **I2C_MST_CTRL** | 0x24 | 8 | 辅助 I²C 主控制器配置（XDA/XCL） |
| **USER_CTRL** | 0x6A | 8 | bit5=使能辅助 I²C，bit2=复位 DMP |

### I²C 通信时序

MPU6050 作为 **I²C 从机**，MCU 作为主机的典型读写时序：

**写寄存器：**
```
START | SLAVE_ADDR(W) | REG_ADDR | DATA | STOP
```

**读寄存器（多字节）：**
```
START | SLAVE_ADDR(W) | REG_ADDR | RESTART | SLAVE_ADDR(R) | DATA[0..N] | STOP
```

对应的 HAL 函数：
```c
// 写单寄存器
HAL_I2C_Mem_Write(&hi2c2, 0xD0, reg, I2C_MEMADD_SIZE_8BIT, &data, 1, 100);

// 读多寄存器
HAL_I2C_Mem_Read(&hi2c2, 0xD1, reg, I2C_MEMADD_SIZE_8BIT, data, len, 100);
```

注意 HAL 库的参数是 **8 位左移后的地址**（即 7 位地址 << 1），写地址 = 0x68<<1 = 0xD0，读地址 = 0xD1。

### 姿态角计算原理

本项目不从 DMP 获取姿态，而是**通过加速度计直接计算**：

```
pitch = atan2(ay, sqrt(ax² + az²)) × 57.3   // 俯仰角
roll  = atan2(-ax, az) × 57.3                // 横滚角
```

- **pitch（俯仰）**：X 轴与水平面的夹角，正值 = 抬头
- **roll（横滚）**：Y 轴与水平面的夹角，正值 = 右侧下沉
- 陀螺仪数据仅作展示，未参与姿态计算（无传感器融合）

> 仅用加速度计计算姿态，在静态时精度高，运动时受线性加速度干扰。高精度场景建议使用 DMP 或互补滤波 / Mahony / Madgwick 等融合算法。

### 常见问题排查

| 现象 | 可能原因 | 检查方法 |
|------|----------|----------|
| WHO_AM_I 返回 0xFF | I²C 总线故障（SDA 一直被拉高） | 检查 SDA/SCL 接线，确认上拉电阻 |
| WHO_AM_I 返回 0x00 | 模块未供电或 I²C 地址错误 | 测 VCC 电压，确认 AD0 电平 |
| 加速度 Z 轴 ≈0 而不是 ≈9.81 | 模块未水平放置 | 平放模块重新测试 |
| 陀螺仪数据全为 0 | 模块静止时角速度确为 0，属正常现象 | 旋转模块观察数据变化 |
| 姿态圆点跳动 | 加速度计受振动干扰 | 增大 DLPF 带宽（CONFIG 寄存器） |
| 读数时有时无 | 杜邦线接触不良或上拉缺失 | 换短线、确认有 4.7kΩ 上拉 |

## 验证标准

- 初始化时 WHO_AM_I 读取返回 0x68
- 平放时 Pitch 和 Roll 接近 0°
- 转动模块时角度值平滑变化
- OLED 显示传感器数据正常（与 Phase 1 显示不冲突）
- IMU 页面（第 5 页）显示加速度、角速度、欧拉角

### 常见问题

| 问题 | 检查项 |
|------|--------|
| WHO_AM_I 返回 0xFF 或 0x00 | PB10/PB11 接线、上拉电阻、AD0 引脚是否接地 |
| 数据全为 0 | PWR_MGMT_1 是否写入 0x00（唤醒） |
| OLED 和 MPU6050 同时不工作 | 各自 I2C 总线独立，分别排查 |
| 只有 OLED 工作 | 检查 PB10/PB11 接线，I2C2 时钟是否使能 |
| IMU 页面显示 NOT FOUND | 检查 MPU6050 模块供电和 I2C2 连接 |
